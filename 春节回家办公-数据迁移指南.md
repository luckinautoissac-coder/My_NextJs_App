# 春节回家办公 - 数据迁移指南

## 📋 概述

这个指南将帮助您在春节回家前，安全地将公司电脑上的所有工作记录导出，并在家里的电脑上恢复，让您可以无缝继续办公。

## ✅ 包含的数据

导出的备份文件将包含以下所有数据：

- ✉️ **所有聊天消息和对话历史** - 您和AI的所有对话记录
- 🤖 **智能体配置** - 您创建或使用的所有智能体设置
- 📁 **话题分组** - 所有话题的组织结构
- ⚙️ **模型配置** - 您选择的AI模型和相关设置（不包含API Key）
- 📚 **知识库** - 所有上传的文档和知识库内容
- ⚡ **快捷短语** - 您自定义的所有快捷短语

> **安全提示**：备份文件**不包含API Key**，这是出于安全考虑。您需要在家里的电脑上重新输入API Key。

---

## 🏢 第一步：在公司电脑导出数据

### 操作步骤

1. **打开应用**
   - 启动您的定制商业助手应用

2. **打开数据设置**
   - 点击左侧边栏底部的 **"⚙️ 设置"** 按钮
   - 在弹出的菜单中选择 **"💾 数据设置"**

3. **查看数据概览**
   - 对话框顶部会显示当前的数据统计：
     - 消息条数
     - 智能体数量
     - 话题数量
     - 已添加模型数量
     - 知识库数量
     - 快捷短语数量
   - 确认这些数据是您想要备份的

4. **导出备份文件**
   - 在 **"数据备份"** 区域，点击 **"导出备份文件"** 按钮
   - 系统会自动下载一个 JSON 文件
   - 文件名格式：`chat-backup-2026-02-13.json`（日期是导出当天）

5. **保存备份文件**
   - 将下载的备份文件保存到安全的位置
   - 建议保存多份：
     - 📱 发送到您的手机（通过微信、邮件等）
     - ☁️ 上传到云盘（百度网盘、OneDrive等）
     - 💾 复制到U盘
     - 📧 发送到您的个人邮箱

> **重要提醒**：请妥善保管备份文件，因为它包含了您的所有工作记录！

---

## 🏠 第二步：在家里电脑恢复数据

### 前置准备

1. **安装应用**
   - 在家里的电脑上安装相同版本的定制商业助手应用
   - 如需安装帮助，请参考原始安装文档

2. **获取备份文件**
   - 从云盘、邮箱或U盘中获取您在公司导出的备份文件
   - 将文件保存到桌面或容易找到的位置

### 恢复步骤

1. **打开应用**
   - 启动家里电脑上的定制商业助手应用

2. **重新配置API Key**
   - 首先需要设置API Key（因为备份不包含）
   - 点击 **"⚙️ 设置"** → **"🔧 API设置"**
   - 输入您的API Key和相关配置
   - 点击保存

3. **打开数据设置**
   - 点击左侧边栏底部的 **"⚙️ 设置"** 按钮
   - 在弹出的菜单中选择 **"💾 数据设置"**

4. **导入备份文件**
   - 在 **"数据恢复"** 区域，点击 **"选择备份文件恢复"** 按钮
   - 在文件选择对话框中，找到并选择您的备份文件
   - 系统会显示即将导入的数据统计信息：
     ```
     即将导入以下数据：
     • XX 条消息
     • XX 个话题
     • XX 个知识库
     • XX 条快捷短语
     • 应用设置（不包括API Key）
     
     ⚠️ 这将覆盖当前的所有数据，是否确认导入？
     ```

5. **确认导入**
   - 仔细核对数据数量是否正确
   - 点击 **"确定"** 按钮开始导入
   - 等待导入完成（通常几秒钟）

6. **验证数据**
   - 导入成功后，会显示成功提示
   - 检查您的：
     - 聊天记录是否都在
     - 话题分组是否完整
     - 知识库文档是否正常
     - 快捷短语是否可用

## 🎯 常见问题

### Q1: 备份文件丢失了怎么办？

**答**：如果没有备份文件，无法恢复数据。建议：
- 在公司电脑还能访问时，重新导出一次
- 以后养成定期备份的习惯（建议每周备份一次）

### Q2: 导入后为什么没有显示聊天记录？

**答**：可能的原因：
1. 备份文件版本不兼容 - 确保家里和公司的应用版本相同
2. 导入过程中出错 - 查看浏览器控制台是否有错误信息
3. 需要刷新页面 - 尝试按 `F5` 刷新页面

### Q3: API Key 为什么没有一起导入？

**答**：出于安全考虑，API Key 不会包含在备份文件中。因为：
- 备份文件可能通过不安全的渠道传输（邮件、云盘等）
- API Key 泄露可能导致费用损失
- 建议使用密码管理器单独保存 API Key

### Q4: 可以在多台电脑之间同步数据吗？

**答**：可以通过手动导出/导入实现：
1. 在电脑A导出备份
2. 在电脑B导入备份
3. 工作一段时间后，在电脑B导出新备份
4. 在电脑A导入新备份

> **注意**：这不是实时同步，而是手动单向同步。

### Q5: 备份文件可以长期保存吗？

**答**：可以，建议：
- 定期备份（每周或每月）
- 保留多个版本（以防某个版本损坏）
- 使用云盘自动备份
- 给文件命名加上日期，方便识别

### Q6: 导入会覆盖当前数据吗？

**答**：是的，导入操作会**完全覆盖**当前所有数据。建议：
- 在导入前，先导出当前数据作为备份
- 确认导入的文件是正确的版本
- 在测试环境先试验一次

## 📝 最佳实践

### 定期备份策略

建议建立以下备份习惯：

1. **每周备份**
   - 每周五下班前导出一次备份
   - 保存到云盘的 `工作备份/周备份/` 目录

2. **重要节点备份**
   - 完成重要项目后立即备份
   - 在出差、休假前备份
   - 系统升级前备份

3. **多重保存**
   - 本地硬盘保存一份
   - 云盘保存一份
   - 邮箱保存一份（重要备份）

### 文件命名建议

使用清晰的命名方式，例如：
```
chat-backup-2026-02-13-公司.json
chat-backup-2026-02-20-家里.json
chat-backup-2026-02-27-重要项目完成.json
```

## 🆘 遇到问题？

如果在导出或导入过程中遇到任何问题：

1. **查看错误提示**
   - 应用会显示具体的错误信息
   - 根据提示进行操作

2. **检查文件完整性**
   - 确保备份文件没有损坏
   - 文件大小不应该是0字节
   - 可以用文本编辑器打开查看是否是有效的JSON格式

3. **浏览器兼容性**
   - 建议使用Chrome、Edge或Firefox浏览器
   - 确保浏览器版本是最新的

4. **联系技术支持**
   - 如果问题仍无法解决
   - 请联系技术支持团队
   - 提供详细的错误信息和截图
